# 🩺 Diabetes Prediction

Проект посвящён задаче бинарной классификации наличия диабета по медицинским показателям пациентов. В рамках работы был выполнен анализ данных, предобработка признаков, визуализация структуры данных, а также сравнение нескольких моделей машинного обучения.

---

## Описание проекта

В качестве исходных данных использовался набор `diabetes.csv`, содержащий медицинские показатели пациентов и целевую переменную `Outcome`.

Цель проекта — построить модель, способную по набору признаков определить вероятность наличия диабета, и сравнить качество различных алгоритмов классификации.

## Данные

Датасет содержит следующие признаки:

- `Pregnancies`
- `Glucose`
- `BloodPressure`
- `SkinThickness`
- `Insulin`
- `BMI`
- `DiabetesPedigreeFunction`
- `Age`

Целевая переменная:

- `Outcome` — наличие диабета (`1`) или его отсутствие (`0`).

---

## Что было сделано

В рамках проекта были выполнены следующие этапы:

- загружен и изучен исходный датасет;
- просмотрены размерность данных, типы столбцов и описательная статистика;
- построены гистограммы распределений признаков;
- обнаружены нули в некоторых медицинских столбцах, где они не соответствуют реальным значениям;
- выполнена замена нулей на `NaN` в признаках `Glucose`, `BloodPressure`, `SkinThickness`, `Insulin`, `BMI`;
- пропущенные значения заполнены медианой соответствующего признака;
- признаки стандартизированы с помощью `StandardScaler`;
- выполнено понижение размерности с помощью `PCA` для визуального анализа структуры данных;
- построены scatterplot-диаграммы для оценки разделимости классов;
- исследована корреляция признаков с целевой переменной;
- обучены и сравнены несколько моделей классификации;
- подобраны гиперпараметры с помощью `GridSearchCV` и `StratifiedKFold`.

---

## Используемые методы

### StandardScaler
Использовался для приведения признаков к единому масштабу. Это особенно важно для моделей, чувствительных к размеру признаков, например `LogisticRegression`.

### PCA
Применялся для уменьшения размерности данных и наглядного отображения структуры признаков в двумерном пространстве.

### GridSearchCV
Использовался для подбора оптимальных гиперпараметров моделей.

### StratifiedKFold
Применялся для кросс-валидации с сохранением пропорций классов в каждом фолде.

### Метрики качества
Для оценки моделей использовались:

- **Accuracy** — доля правильных предсказаний;
- **F1-score** — баланс между precision и recall, особенно полезен при несбалансированных классах.

## Модели, которые были обучены

В проекте были протестированы следующие алгоритмы:

- `LogisticRegression`
- `DecisionTreeClassifier`
- `RandomForestClassifier`
- `XGBClassifier`

Для каждой модели был выполнен подбор параметров, а затем проведена оценка на тестовой выборке.

## Результаты

| Модель | Accuracy | F1-score |
|--------|----------|----------|
| LogisticRegression | 0.7078 | 0.6154 |

## Выводы

По результатам экспериментов можно сделать следующие выводы:

- линейная модель `LogisticRegression` показала результат ниже, чем ансамблевые и древовидные методы;
- `DecisionTree`, `RandomForest` и `XGBoost` лучше справились с задачей, что говорит о наличии нелинейных зависимостей в данных;
- наибольшее значение `accuracy` показал `XGBoost`;
- наибольшее значение `F1-score` получила модель `DecisionTree`.

Таким образом, для данного набора данных более подходящими оказались нелинейные модели машинного обучения.
